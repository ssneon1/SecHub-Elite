⚖️ ITSAR / 3GPP Regulatory Context

DDoS protection is not just an IT choice; it's a mandatory regulatory requirement for modern network elements (NE) like UPF (User Plane Function).

Section 4.2.3.3.1

System handling during overload situations. NE shall have protection mechanisms against Network and Application-level DDoS attacks to deal with overload situations from attack generators or increased traffic.

Section 4.2.3.3.3

Excessive Overload Protection. The system must act predictably. In extreme cases, a controlled system shutdown is preferable to uncontrolled failure of security functions.

ITSAR UPF Requirement 2.8.1

Requirement for limiting RAM per application, maximum sessions, CPU resources per process, and transaction volume per IP to avoid impairment of availability.

🧬 Attack Taxonomy & Lab Simulation

Volume & Protocol Based

  • Random Source Attack: Diff source addresses confuse tracebacks.
  • SMURF Attack: Poisoned ICMP echo requests broadcasted to target.
  • LAND Attack: Src/Dst IP matches target to loop kernel resources.
  • SYN Flood: Half-open attack exhausts TCP handshake table.
# SMURF Attack Example # hping3 --icmp --flood 127.0.0.1 -a 127.0.0.1 # LAND Attack Example # hping3 -S -p 80 127.0.0.1 -a 127.0.0.1

📡 Application Layer Architecture

Architecture Models

  • Thread-based (Apache): Creates new thread/process per request. High resource drain. VULNERABLE to Slowloris.
  • Event-based (Nginx): Uses same thread to handle multiple requests efficiently. Highly resistant.

Slowloris Workflow

  1. Make thousands of partial HTTP requests.
  2. Send headers periodically (~15s) to stay open.
  3. Never close connections. Thread pool exhausted.

🛡️ Mitigation & SOC Operations

Advanced Mitigation

  • ICMP Rate Limiting: Enforced at the OS level.
  • Fingerprint Filtering: Identify specific attack features in UDP data segments.
  • WAF Challenge: Force reCAPTCHA (v2/v3) on suspicious GET requests.
