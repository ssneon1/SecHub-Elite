Attack Taxonomy & Lab Simulation

Volume & Protocol (L3/L4)

  • Random Source Attack: Randomized src IPs confuse tracebacks & ACLs.
  • SMURF Attack: Poisoned ICMP echo requests broadcasted to target.
  • LAND Attack: Src/Dst IP = target IP, loops kernel resources to OOM.
  • SYN Flood: Half-open attack exhausts TCP handshake table (backlog overflow).
  • UDP Fragmentation: Oversized datagrams exhaust IP reassembly buffers.
SMURF / LAND LAB COMMANDS (hping3)
# SMURF Attack
hping3 --icmp --flood 127.0.0.1 -a 127.0.0.1

# LAND Attack
hping3 -S -p 80 127.0.0.1 -a 127.0.0.1

# SYN Flood with random src IP
hping3 -S --flood -V -p 80 --rand-source [TARGET]

Application Layer (L7)

  • Slowloris: Thousands of lingering partial HTTP requests keep threads open.
  • R.U.D.Y: "R U Dead Yet?" - posts files byte-by-byte to exhaust threads.
  • HTTP/2 Rapid Reset: Exploits RST_STREAM burst for CPU spikes (CVE-2023-44487).
  • AI-Driven Bots: Mimic human traffic patterns to bypass legacy WAF rules.
  • Cache Busting: Cache-miss floods origin server by bypassing CDN cache.

2. Send headers every ~15s to keep alive
3. Never complete the request

Application Layer Architecture

Server Architecture Models

  • Thread-based (Apache): Creates new thread/process per request. High resource drain. VULNERABLE to Slowloris
  • Event-based (Nginx): Same thread handles multiple requests via event loop. RESISTANT
  • Async (Node.js/Go): Non-blocking I/O naturally limits thread exhaustion.

Defense & SOC Operations

  • ICMP Rate Limiting: Enforced at OS/firewall level to cap flood volume.
  • Fingerprint Filtering: Identify specific attack signatures in UDP data segments.
  • WAF Challenge: reCAPTCHA v2/v3 on suspicious GET/POST spikes.
  • Anycast Routing: Spread incoming flood across global POP nodes.

ITSAR / 3GPP Regulatory Context

DDoS protection is not just an IT choice — it is a mandatory regulatory requirement for modern network elements (NE) like the UPF (User Plane Function) in 5G Core.

3GPP § 4.2.3.3.1 — Overload Protection

System handling during overload situations. NE shall have protection mechanisms against Network and Application-level DDoS attacks to deal with overload situations from attack generators or increased traffic.

ITSAR UPF Req 2.8.1 — Resource Limits

Requirement for limiting RAM per application, maximum sessions, CPU resources per process, and transaction volume per IP to avoid impairment of availability. Applies directly to 5G UPF components.

3GPP § 4.2.3.3.3 — Controlled Degradation

Excessive Overload Protection. The system must act predictably. In extreme cases, a controlled system shutdown is preferable to uncontrolled failure of security functions.

Network Slicing Security — 5G NSA/SA

DDoS protection must be slice-aware. An attack on one slice (e.g., IoT/mMTC) must not impair the availability of mission-critical slices (uRLLC) such as autonomous vehicles or industrial control systems.
